# Architecture

## What it does
The system performs a high-performance, semi-semantic movie search by combining vector similarity search (FAISS) with metadata-driven popularity weighting to improve result relevance without significant latency overhead.

## Components
- **Vector Retrieval**: A FAISS index provides initial candidate retrieval based on user query embeddings.
- **Candidate Pool**: The system retrieves a fixed pool of 200 candidates to balance coverage and performance.
- **Popularity Bias**: A logarithmic popularity boost ($log(1 + vote\_count)$) is applied to the retrieved candidates.
- **Scoring Engine**: A hybrid score is calculated as `score = -L2_distance + (0.5 * log_popularity)`.
- **Ranking**: The final list is generated by sorting candidates by the hybrid score and returning the top-k results.

## Why it works
The architecture addresses the "cold start" or "niche" problem in purely semantic search. By applying a logarithmic transformation to `vote_count`, the system incorporates social proof (popularity) into the ranking without allowing extreme outliers (blockbusters) to completely eclipse relevant niche matches. Subtracting the L2 distance (which is higher for worse matches) aligns the vector similarity with a "higher-is-better" score, facilitating simple integration with metadata signals.

## Tradeoffs
- **Hybrid Complexity**: The system abandons pure vector search for a heuristics-based reranking. While this improves user satisfaction, it introduces a manual weighting factor (0.5) that may need recalibration if the dataset distribution shifts significantly.
- **Candidate Pool Limitation**: By limiting retrieval to 200 candidates, the system ensures sub-15ms latency but potentially sacrifices recall for very long-tail, obscure queries.

## Key experiments
- **Candidate Pool Expansion (Exp #8)**: Demonstrated that increasing the pool size to 100 significantly improved recall compared to smaller baseline pools.
- **Logarithmic Popularity Boosting**: The winning strategy involved applying $log(1 + vote\_count)$ to normalize popularity signals, effectively filtering noise from raw vote counts.
- **Hybrid Scoring**: Experiments with purely semantic search (FAISS) showed high latency but lower-than-desired user relevance; the integration of popularity metadata bridged this gap.
